# 酒馆冒险 (Tavern Adventure)

AI 驱动的奇幻文字冒险 + 本地 D20 回合制战斗 + 动态天空与天气视觉。

## 体验

https://minanoneko.github.io/tavern-adventure/

## 启动

```bash
npm install
npm run dev
```

浏览器打开 `http://localhost:5173`

## 玩法

创建角色（11 个职业 + 年龄可选）→ AI 生成开局剧情 → 自由探索。

- **固定按钮**：交谈/调查/休息/商店
- **自定义输入**：输入行动意图（如"绕到后门看看""悄悄跟踪那个人"），AI 回应对应剧情
- **不能口胡**：不能写"我捡到 100 金币""我生成一条龙""我一刀秒杀"，系统会拦截

### 战斗

AI 触发战斗（`combatStart`），或玩家输入攻击指令后进入战斗。

回合制 D20 系统：

- **攻击** — 武器骰 + 属性修正 vs 敌人 AC（含完整算式日志）
- **技能** — 消耗 MP，配置伤害骰 + 属性倍率，有投骰滚动动画
- **物品** — 治疗药水、燃烧瓶、烟雾弹
- **防御** — 伤害减半
- **逃跑** — d20 + 敏修正 vs DC14

战斗奖励本地结算：经验、钱币、材料、稀有饰品。

战斗结束后 AI 自动获得战斗结果摘要，剧情不断裂。

### BOSS

只在关键剧情节点触发（区域深处、任务终点、反派登场），有独立掉落。需 `worldState` 中存在合法 flag 才会生成。

### 地图与场景过渡

- **不可瞬移**：玩家必须发送明确的移动指令（如"向北走""进入城堡"）才能切换场景
- **区域发现**：探索过的区域会记录在记忆中，可通过地图查看
- **区域解锁**：部分区域需要前置条件（任务、Boss 击杀、特殊物品）

### storyHooks（剧情钩子）

取代传统任务系统。AI 在剧情中埋设线索和钩子，玩家自由选择是否跟进，不再有"任务完成/失败"的硬性判定。

### CHECK 判定

需要 AI 选项带 `requiresCheck: true` 才会判定。`d20 + 属性修正 vs DC`，成功/失败由本地结算。

### 年龄特质

创建角色时可选择年龄阶段（少年/青年/中年/老年），不同年龄影响初始属性分配。

### 安全开局

AI 生成开场失败时，系统自动降级使用预设的低风险生活化开局（市集纠纷、铁匠铺赝品、小礼拜堂钟声），避免矿洞/失踪商队等老套 TRPG 桥段。

**职业风味开场**：每个职业在开局时获得专属的行动选项和旁白提示（法师→魔力感知、猎人→辨认痕迹、战士→压住场面等），让重复开局也有新鲜感。

### 动态天空与天气

背景渲染分多层（天穹、日月星辰、云层、大气层、天气），根据游戏内时间（清晨/上午/中午/下午/黄昏/夜晚）和天气（晴/多云/雾/雨/雪/暴风雨）实时变换颜色、光照和特效。天气变化趋势（转晴/恶化/暴风雨酝酿等）也会影响大气层效果。

### 装备稀有度色彩

装备按稀有度着色：普通→米白、罕见→绿色、稀有→粉色、史诗→紫色、传说→金色、诅咒→红色、遗物→青色。直观反映物品品质。

### 道具目录

物品使用白名单机制（`itemCatalog.ts`），只能购买、使用目录内的合法物品，防止非法物品混入背包。

## 技术栈

React + TypeScript + Vite + Zustand + Tailwind CSS + Zod

```
src/
├── components/     UI 组件
├── services/      核心逻辑 (combat/gameEngine/judgeService)
├── store/         Zustand 状态管理
├── types/         类型定义
├── prompts/       AI 提示词
├── data/          技能/装备/地图数据
└── utils/         工具函数 (equipmentRules/skillRules/weaponDamage)
```

## 配置

复制 `.env.example` 为 `.env`：

```
VITE_USE_MOCK_AI=true    # 本地模拟，不开战
VITE_DEBUG_PROMPT=false
```

如需 AI：设为 `false`，填入 API 地址、模型、Key。

### API Key 安全说明

- **纯前端项目无法隐藏 API Key**。浏览器端发起的请求，Key 始终暴露在客户端。
- 公开部署时不内置任何作者自己的付费 Key。用户**只填写自己的 Key**，风险由本地环境承担。
- Key 默认仅保存在 sessionStorage（关浏览器即清空），设置页可选 local 持久化（泄露风险）。
- 部分 API 不支持浏览器直连（CORS 限制），请确认服务商支持浏览器端调用。
